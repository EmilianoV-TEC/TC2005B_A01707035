Preguntas lab 1

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • El internet se refiere a una red de computadoras que existe alrededor del mundo, mientras que la World Wide Web indica la red de páginas que se encuentran en el internet

  • ¿Cuáles son las partes de un URL?

    • El protocolo, el dominio, la ruta y los parámetros

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Solicitar datos de un servidor
    HEAD: Permite revisar si cierta información está disponible en el servidor
    POST: Enviar datos a un servidor
    PUT: Actualiza cierta información o recurso publicado en el servidor
    PATCH: Permite modificar un recurso
    DELETE: Para borrar un recurso en el servidor

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • El método POST ya que este se encarga de enviar datos al servidor y no modifica el URL con información importante

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El método GET ya que se está solicitando la información del sitio que se va a desplegar

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Es un OK, significando que se proceso la acción correctamente, no hay error

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Dependería, ya que si es una url incorrecta ingresada por el usuario es su error, si es alguna redirección sería culpa del desarrollador

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Si, porque es un problema que ocurre en el servidor

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Significa que ese atributo ya no recibe mantenimiento o seguimiento, pero aún puede ser utilizado. Algunos ejemplos son center y u

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • HTML5 incluye más funciones que HTML4, además de que es un conjunto de tecnologías, mientras que HTML4 mezcla los diferentes aspectos

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • tr, th, tr, td, thead, tbody, tfoot

  • ¿Cuáles son los principales controles de una forma HTML5?

    • input, label, button, textarea

  • ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/

    • Con Google Chrome, obtiene un puntaje de 476

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Planificación, análisis, diseño, implementación, mantenimiento

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • Planificación, análisis, diseño, implementación, pruebas, despliegue, mantenimiento

Preguntas lab 2

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Debe de ser usado cuando es necesario sobreescribir otros atributos con el objetivo de mantener alguno, como cuando se sabe de un aspecto importante para e diseño, y no se quiere que cambie a pesar del uso de otras clases o id

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Por las dimensiones, ya que si no concuerda con las del sitio o dispositivo es posible que no se vea como se espera. También se deben cuidar las rutas

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Creo que px y % sirven bien para las páginas, el porcentaje permite mantener imágenes o figuras de con un tamaño adecuado con respecto a las dimensiones, y px puede ayudar a mantener la misma visibilidad

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Debido a que incluye una menor cantidad de caracteres, haciendo que leerlo sea más rápido
